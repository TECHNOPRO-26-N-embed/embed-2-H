## 3. 機能要件

### 3-1. このガジェットができること（必須機能）
<!-- > ※ 「〜できる」の形で箇条書き。最低3つ書いてください。 -->

- ドからシまでの音と1-7の番号が対応しており、各番号ごとに異なる音（周波数）を鳴らせる（対応表は下記参照）。基準オクターブを0として、`*`で-1オクターブ、`#`で+1オクターブに切替できる
- 入力された複数の音程（最大20音）を記録し、それが再入力と一致しているか確かめられる。オリジナル側のみ、入力完了を keypad「0」操作で確定できる
- マネ側の入力時に、入力された音程を1音ごとに即時比較できる。1ミスでその時点で不正解とし、LEDライト（赤）を3回点滅させる。全音一致した場合は自作したクリア音（休符を含むメロディ）を鳴らせる。マネ側は最終入力から制限時間以内に次入力できなければ不正解にできる
- モード選択後のみ`A/B/C`入力を受け付け、distance値と難易度を変更できる（A:易しい, B:標準, C:難しい）
- デフォルト曲（基本曲）を再生中は、ミミック入力を含むすべての入力を受け付けない
- 状態遷移が発生したとき、シリアルモニタに「遷移元 -> 遷移先」と遷移理由を表示できる

### 3-2. あったらいい機能（任意・余裕があれば）

- 直近5ターンの正解率を算出し、シリアルモニタに表示できる
- LCD1602（パラレル接続）に連続正解ターン数や何音目かを分数で表示できる
- 難易度（入力上限音数、入力制限時間、テンポ）をゲーム開始時に切り替えできる
- オリジナル入力待機中にボタンでデフォルト曲モードへ切り替え、オリジナル入力なしでマネ入力フェーズを開始できる

### 3-3. やらないこと（スコープ外）
<!-- > ※ 「〜はしない」で書いてください。AIが勝手に膨らませる機能を防ぐために重要です。 -->

- 同時複数入力はしない

### 3-4. 非機能要件（任意）
<!-- > ※ 余裕があれば書いてください。AIはここを無視しがちです。 -->

- **安全性**：通常の音入力は1-7のみ有効とし、その他は無効として無視する（ただし`0/*/#/A/B/C`は仕様で定義した条件下のみ受理）
- **応答時間**：入力から音出力までの許容時間は100ms以内とし、判定結果表示までの許容時間300msとする
- **入力安定性**：キー入力・モード切替ボタン入力はチャタリング対策を行い、押しっぱなしによる連続誤入力を防止する

## 4. 使用部品リスト

### 4-1. グループ共通部品（グループ全員で使うもの・1つ）

**共通部品：Passive Buzzer（受動ブザー）**

**選定理由：該当処理ごとに音を鳴らすことができ、汎用性があるため**

### 4-2. 自分が追加で使う予定の部品（ELEGOOキットより）

| 部品名 | 何のために使うか（機能との対応） |
|:---|:---|
| Membrane Switch Module（膜スイッチモジュール） | ドからシ（1-7）入力、オクターブ変更（`*`/`#`）、難易度変更（`A/B/C`） |
| Button（プッシュボタン） | デフォルト曲モード切替 |
| Passive Buzzer（受動ブザー） | 音を鳴らす |

膜スイッチモジュールは、D9-D2（デジタルピン）を使用する。音入力は1-7を使用し、0はオリジナル入力確定、`*`と`#`はオクターブ切替、`A/B/C`はモード選択後の難易度入力に使用する
ボタンはD12ピンに接続し、デフォルト曲モード切替に使用する
受動ブザーはD10ピンに接続する
LEDライト（赤）はD11ピンに接続する

| 機能 | Arduinoピン | 備考 |
|:--|:--|:--|
| Membrane Switch Module 1 | D9 | ド |
| Membrane Switch Module 2 | D8 | レ |
| Membrane Switch Module 3 | D7 | ミ |
| Membrane Switch Module 4 | D6 | ファ |
| Membrane Switch Module 5 | D5 | ソ |
| Membrane Switch Module 6 | D4 | ラ |
| Membrane Switch Module 7 | D3 | シ |
| Membrane Switch Module 8 | D2 | 予備（音入力には使わない） |
| Button | D12 | デフォルト曲モード切替（INPUT_PULLUP想定） |
| Passive Buzzer | D10 | 音出力 |
| LEDライト（赤） | D11 | 不正解時点滅 |

## 5. AIレビュー記録

<!-- > ※ 下記の質問をAIに投げ、回答と自分の対応を記録してください。 -->

### Q1: 不足の確認
<!-- > 「この要件定義書をもとに基本設計を作ろうとしたとき、不足している情報や曖昧な点はどこですか？」 -->

AIの回答（要約）：
- システム全体の動作フローが不適切
- Membrane Switch Moduleのどのボタンと音が対応するのかが明記されていない
- 音の出力仕様
- 1ゲームで入力できる音数上限
- 応答時間が空欄
- 各部品が使用するピンの詳細

対応した内容：
- 1ターンの流れを　入力 -> 記録 -> 再入力 -> 判定 -> 結果 -> 入力に戻る　とした
- ドからシがMembrane Switch Moduleの1-7に対応し、`*`/`#`でオクターブ切替できるよう表記した
- Passive Buzzerのパラメータで、ヘッダーファイルに記載された数値をもとに音を定め鳴らす
- UNO R3のメモリ上限が不明なため、音の入力上限数は20音とする。
- ボタンを押してからの音を出力とそれが一致しているかの判定を100ms以内とする

### Q2: 実現可能性の確認
<!-- > 「この要件はArduino UNO R3とELEGOOキットで実現できますか？難しい要件があれば教えてください。」 -->

AIの回答（要約）：
- 今の要件なら、Arduino UNO R3とELEGOOキットで十分可能
- キーパッド入力の安定化　チャタリングや押しっぱなしによる誤入力
- 不正解の判定を1音ミスにするか、最後まで入力後にするのか
- toneによる鳴らしている間の入力受付の有無　ノンブロッキング化（millisベース）
- LCD1602を並列接続するとピン消費が増え、配線が複雑になる


対応した内容（スコープ変更など）：
- キーパッドの入力の仕様について調べる
- 1音ミスで不正解に変更
