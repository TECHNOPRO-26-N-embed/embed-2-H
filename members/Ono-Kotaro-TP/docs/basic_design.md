## 1. システム全体像

### 1-1. ブロック構成（入力 → 処理 → 出力）

<!-- > ※ Arduinoを中心に「何が入力で何が出力か」をテキストの図で整理します。 -->

<!--
【入力】                 【Arduino UNO R3】            【出力】
タクトスイッチ ─────→┐                         ┌──→ LED（赤・緑）
超音波センサー ─────→┤   状態管理・判定処理    ├──→ パッシブブザー
                      └─────────────────────────┘
-->

```
【入力】Membrane Switch Module（1-7と0, `*`, `#`, `A/B/C`, D9-D2）、Button（D12: 画面遷移の確定）
  ─────────────→ 【Arduino UNO R3】入力受付・音程記録・一致判定
          ─────────────→ 【出力】Passive Buzzer（D10, 音の再生・結果通知）、LEDライト（D11, 不正解時点滅）
```

補足説明（図で表現しきれない場合）：
- 通常の音入力は1-7（D9=1, D8=2, ..., D3=7）を有効値とし、それ以外は無効として無視する（ただし `0/*/#/A/B/C` は仕様に従って扱う）。
- keypad「0」はモード切替専用とし、押すたびに「オリジナル/既存曲」をトグルする。
- D12ボタンは確定ボタンとして使い、モード確定・難易度確定・オリジナル開始・オリジナル終了に使う。
- `*` は -1オクターブ、`#` は +1オクターブ切替として、オリジナル/マネ入力中のみ受理する。
- `A/B/C` はモード選択後のみ受理し、distance値（A=30, B=20, C=10）と難易度を更新する。
- ゲーム1ターンで保存できる音数は最大20音。
- 要件定義書 3-3 に基づき、同時複数入力は扱わない。

### 1-2. 状態遷移（ガジェットの「ふるまい」設計）

<!-- > ※ ガジェットが「どんな状態を持つか」を定義します。
> この設計が loop() の骨格になります。 -->

<!--
[電源ON / 初期化]
  ↓（初期化完了）
[待機中] ──（センサー検知）──→ [計測中] ──（閾値超え）──→ [警告中]
   ↑                                                            │
   └────────────────────（リセットボタン）────────────────────────┘
-->

```
[電源ON / 初期化]
  ↓
[モード選択]
  ├─（keypad「0」でオリジナル/既存曲を切替、都度シリアル表示）
  └─（D12押下で確定）→ [難易度選択]
                          ├─（A/B/Cで難易度選択、都度シリアル表示）
                          └─（D12押下で確定）
                                 ├─（オリジナルモード）→ [オリジナル開始待機] ─（D12）→ [オリジナル入力記録] ─（D12で終了確定）→ [マネ入力待機]
                                 └─（既存曲モード）→ [既存曲再生] ─（再生中は全入力無効）→ [マネ入力待機]

[マネ入力待機]
  ├─（全一致）→ [正解通知] → [モード選択]
  └─（1ミス/タイムアウト）→ [不正解通知] → [モード選択]
```

**各状態の意味：**

| 状態名 | この状態のとき、何をしているか | LEDやブザーなど出力の様子 |
|:--|:--|:--|
| モード選択 | keypad「0」でモードをトグルし、D12で確定する | 現在モードをシリアル表示 |
| 難易度選択 | `A/B/C`で難易度を選び、D12で確定する | 現在難易度をシリアル表示 |
| オリジナル入力待機 | オリジナル入力の開始ボタン待ち | なし |
| オリジナル入力記録 | 入力された音程を記録する | 入力に対応する音を鳴らす（`*`/`#`でオクターブ変更可能） |
| 既存曲再生 | 既存曲を再生し、再生中は入力を受け付けない | 既存曲を再生 |
| マネ入力待機 | マネ側のキー入力を待つ | 入力に対応する音を鳴らす（`*`/`#`でオクターブ変更可能） |
| 正解通知 | 全入力一致時の結果通知 | 休符を含む自作クリア音を鳴らす |
| 不正解通知 | 1ミス時の結果通知 | LEDライト（D11）を0.2秒×3回点滅 |

## 2. ソフトウェア設計

### 2-1. データ設計（使う変数・定数の洗い出し）

<!-- > ※ プログラムで扱う「情報の種類」を先に整理します。 -->

<!--
| 情報の名前 | 何を表すか | 型（イメージ） | バイト数 | 初期値 | 備考 |
|:--|:--|:--|:--|:--|:--|

| （例）lastMillis | millis()用タイマー | unsigned long | 4B | 0 | |
-->

| 情報の名前 | 何を表すか | 型（イメージ） | バイト数 | 初期値 | 備考 |
|:--|:--|:--|:--|:--|:--|
| currentState | 現在状態（待機/記録/判定/通知） | int | 2B | 0 | 状態遷移制御用 |
| inputKey | 入力キー値（1-7、turn切替0、オクターブ`*`/`#`、難易度`A/B/C`） | int | 2B | 0 | 条件に応じて受理 |
| originalSeq | オリジナル入力列 | int型キュー | 最大20 | 空 | 長さ上限20 |
| expectedKey | 比較対象のオリジナル音 | int | 2B | 0 | キューから1音取り出して比較 |
| originalLength | オリジナル入力数 | int | 2B | 0 | 1ターンの正解必要数 |
| mimicCount | マネが正しく入力した音数 | int | 2B | 0 | originalLength 到達で正解 |
| lastMimicInputMs | マネ最終入力時刻 | unsigned long | 4B | 0 | 難易度ごとのタイムアウト判定 |
| isAllMatch | 一致判定結果 | bool | 1B | false | 全一致ならtrue |
| missDetected | 1ミス検出フラグ | bool | 1B | false | 1ミス時点で不正解 |
| compareIndex | マネ比較時の参照位置 | int | 2B | 0 | originalSeqの何番目を比較するか |
| lastDebounceTimeKey | キー入力の前回確定時刻 | unsigned long | 4B | 0 | チャタリング対策用 |
| lastDebounceTimeButton | モード切替ボタンの前回確定時刻 | unsigned long | 4B | 0 | チャタリング対策用 |
| lastStableInputKey | 前回確定したキー入力値 | int | 2B | 0 | 押しっぱなし誤入力対策 |
| lastButtonStableState | 前回確定したボタン状態 | bool | 1B | false | 押しっぱなし誤入力対策 |
| currentMode | 現在モード（オリジナル入力/デフォルト曲） | int | 2B | 0 | モード分岐制御用 |
| octaveShift | 現在のオクターブシフト（-1/0/+1） | int | 2B | 0 | `*`/`#`で更新 |
| distanceValue | 難易度調整用distance値 | int | 2B | 20 | A=30, B=20, C=10 |
| mimicTimeoutMs | 現在難易度の入力制限時間 | unsigned long | 4B | 3000 | A/B/Cで更新 |
| modeSelected | モード選択済みフラグ | bool | 1B | false | true のときのみA/B/C受付 |
| isPlaybackLocked | 曲再生中入力ロックフラグ | bool | 1B | false | true の間は全入力無効 |

処理用定数（実装時に定義）:
- MAX_SEQUENCE_LENGTH = 20
- MIMIC_TIMEOUT_BASE_MS = 3000
- INPUT_TONE_DURATION_MS = 100
- DEFAULT_SONG_TONE_DURATION_MS = 180
- DEFAULT_SONG_GAP_MS = 80
- CLEAR_TUNE_GAP_MS = 90
- WRONG_LED_BLINK_MS = 200
- WRONG_LED_BLINK_COUNT = 3
- DEBOUNCE_DELAY_MS = 50
- ORIGINAL_MODE = 0, MIMIC_MODE = 1
- RESULT_CORRECT = 1, RESULT_WRONG = -1, RESULT_CONTINUE = 0
- MODE_ORIGINAL_INPUT = 0, MODE_DEFAULT_SONG = 1

**SRAM使用量チェック（主要ゲーム制御変数）**:
- グローバル変数の合計: 2B (currentState) + 2B (currentMode) + 2B (inputKey) + 40B (originalSeq) + 2B (expectedKey) + 2B (originalLength) + 2B (mimicCount) + 4B (lastMimicInputMs) + 1B (isAllMatch) + 1B (missDetected) + 2B (compareIndex) + 4B (lastDebounceTimeKey) + 4B (lastDebounceTimeButton) + 2B (lastStableInputKey) + 1B (lastButtonStableState) = **71B**
- 実装コードでは上記に加えて、カバレッジ記録用配列と状態ログ用変数を追加している。
- Arduino UNO R3のSRAM上限（2048B）に対して十分余裕あり。

### 2-3. タイミング設計（delay vs millis）

| 処理 | 必要な周期 | delay / millis | 判断の理由 |
|:--|:--|:--|:--|
| キー入力監視 | 常時 | millis | 入力取りこぼし防止のため |
| 一致判定実行 | 入力ごと | millis | 1ミス不正解を即時判定するため |
| 結果通知音 | 通知時 | delay または millis | 高速に判定する必要はないため |

## 音とピンの対応表・ブザー仕様

| キー番号 | Arduinoピン | 音階 | 周波数例(Hz) |
|:--|:--|:--|:--|
| 1 | D9 | ド | 262 |
| 2 | D8 | レ | 294 |
| 3 | D7 | ミ | 330 |
| 4 | D6 | ファ | 349 |
| 5 | D5 | ソ | 392 |
| 6 | D4 | ラ | 440 |
| 7 | D3 | シ | 494 |
| 8 | D2 | 予備（音入力には使用しない） | - |

Passive Buzzer（D10）は、各番号に対応する音階の周波数で100ms鳴動する。`*`/`#`によりオクターブを-1/0/+1で切り替える。正解時は休符を含む自作クリア音、デフォルト曲再生時は休符を挟んだメロディとして鳴動。不正解時はLEDライト（D11）を0.2秒×3回点滅。

### 3-2. 電気的考慮事項

| 用途 | 判断・計算メモ |
|:--|:--|
| Membrane Switch Module | 電圧範囲: 3.3V〜5V, 通常音入力は1-7のみ有効（仕様キーを除く） |
| Passive Buzzer | 動作電圧: 3.3V〜5V、tone()でキー対応の262〜494Hzを出力。オクターブ切替時は周波数を1/2または2倍に補正 |
| Button | D12でINPUT_PULLUPを使用し、押下時LOWとして判定（モード確定/難易度確定/入力開始・終了確定用） |
| LCD1602A | 動作電圧: 5V、接続方式: パラレル |
| LED電流制限抵抗 | (5V − 2.0V) ÷ 0.02A = 150Ω → 220Ω使用 |
