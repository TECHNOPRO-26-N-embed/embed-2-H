## 1. システム全体像

### 1-1. ブロック図（入力 → 処理 → 出力）

```
【入力】                    【Arduino UNO R3】                【出力】
ボタン(左)  ──────────→                                ┌──→ 4桁ディスプレイ
ボタン(中)  ──────────→   スロット状態管理・乱数生成      ├──→ パッシブブザー
ボタン(右)  ──────────→   ボタン判定・表示更新制御        ┘
```

---

### 1-2. 状態遷移図（システムの「ふるまい」）

```
[電源ON]
	↓ 初期化完了
[待機(初期数字表示)] ─── 任意ボタン押下 ───→ [回転中]
│
├── 左/中央/右ボタン押下（順不同）→ 対応するスロットを停止（[左停止] [中央停止] [右停止]）
└── 3スロットすべて停止したら
	↓
[全停止・判定]
├── アタリ(3桁一致) → [当選音再生]
├── 任意機能ONかつリーチ(2桁一致) → [リーチ演出]
└── 上記以外 → [結果表示待機]

[リーチ演出] ─── リーチ音再生後、任意ボタン押下で未一致1桁を回転開始 → 対応ボタン押下で停止 ───→ [全停止・判定]
[当選音再生] ─── 再生完了 ───→ [結果表示待機]
[結果表示待機] ─── 任意ボタン押下で再スタート ───→ [回転中]
```

**状態の意味を説明してください：**

| 状態名 | この状態のとき、何をしているか |
|:--|:--|
| 待機(初期数字表示) | 電源ON直後。各スロットに0〜9の任意数字を表示し、開始入力を待つ |
| 回転中 | 未停止のスロット数字を50ms周期(0.05秒)で更新する |
| 左停止 | 左スロットを固定|
| 中央停止 | 中央スロットを固定|
| 右停止 | 右スロットを固定|
| 全停止・判定 | 3スロット停止後に一致判定し、アタリ時のみブザーを鳴らす |
| リーチ演出（任意） | 2桁一致時にリーチ音を鳴らし、任意ボタン押下で未一致1桁を回転開始して、対応ボタン押下で停止する |
| 当選音再生 | 3桁一致時の当選音を再生する |
| 結果表示待機 | 判定結果を表示したまま、次ゲーム開始入力を待つ |

## 2. ソフトウェア設計

### 2-1. データ設計（使う変数・定数の洗い出し）

> 要件定義書の機能（3スロット、3ボタン、50ms更新、当選/リーチ音、再スタート）を満たすための主要データを定義する。

| 情報の名前 | 何を表すか | 型（イメージ） | バイト数 | 初期値 | 備考 |
|:--|:--|:--|--:|:--|:--|
| slotDigits[3] | 左・中・右スロットの現在数字 | uint8_t配列 | 3 | {0,0,0} | 0〜9を保持 |
| slotStopped[3] | 各スロットが停止済みか | bool配列 | 3 | {false,false,false} | 押下された対応ボタンの桁を順不同でtrue化 |
| currentState | ゲーム状態（待機/回転/停止判定/結果） | enum class SlotState : uint8_t | 1 | STATE_WAIT | 状態遷移で使用 |
| isSpinning | 現在回転中か | bool | 1 | false | 全桁停止時にfalse |
| stoppedCount | 停止した桁数 | uint8_t | 1 | 0 | 0〜3 |
| lastSpinUpdateMs | 前回の数字更新時刻 | uint32_t | 4 | 0 | millis基準 |
| spinIntervalMs | 数字更新周期(50ms) | constexpr uint16_t | 0 | 50 | constexprでSRAM未使用 |
| lastDebounceMs[3] | 各ボタンの最終デバウンス時刻 | uint32_t配列 | 12 | {0,0,0} | チャタリング対策 |
| debounceIntervalMs | デバウンス時間 | constexpr uint16_t | 0 | 50 | constexprでSRAM未使用 |
| prevButtonLevel[3] | 前回のボタン入力レベル | uint8_t配列 | 3 | {HIGH,HIGH,HIGH} | INPUT_PULLUP前提 |
| stableButtonLevel[3] | 確定したボタン入力レベル | uint8_t配列 | 3 | {HIGH,HIGH,HIGH} | エッジ検出に使用 |
| reachEnabled | リーチ演出機能ON/OFF | bool | 1 | true | 任意機能 |
| isReach | 現在リーチ成立中か | bool | 1 | false | 2桁一致時にtrue |
| isReachSpinActive | リーチ演出で未一致桁の回転開始済みか | bool | 1 | false | 任意ボタン押下でtrue |
| reachSlotIndex | リーチ時に未一致の桁番号 | int8_t | 1 | -1 | 0:左,1:中,2:右 |
| hitFlag | 3桁一致（当選）判定結果 | bool | 1 | false | 当選音再生の条件 |
| hitCount | 当選回数 | uint16_t | 2 | 0 | 統計表示用（任意） |
| playCount | プレイ回数 | uint16_t | 2 | 0 | 統計表示用（任意） |
| btnPins[3] | 左・中・右ボタンのピン番号 | constexpr uint8_t配列 | 0 | {4,5,6} | constexprでSRAM未使用 |
| buzzerPin | ブザー接続ピン番号 | constexpr uint8_t | 0 | 9 | constexprでSRAM未使用 |
| displayClkPin | 4桁ディスプレイCLKピン番号 | constexpr uint8_t | 0 | 2 | constexprでSRAM未使用 |
| displayDioPin | 4桁ディスプレイDIOピン番号 | constexpr uint8_t | 0 | 3 | constexprでSRAM未使用 |

> [!CAUTION]
> **SRAM使用量チェック（Arduino UNO R3 の上限は 2048B）**
>
> グローバル変数の合計（上表のSRAM使用分）: **40 B**
>
> 計算: 3+3+1+1+1+4+12+3+3+1+1+1+1+1+2+2 = 40
>
> | 合計バイト数 | 判定 |
> |:--|:--|
> | 40B / 2048B（約2.0%） | ✅ 余裕あり |
>
> 備考: 上記は「設計上の主要グローバル変数」の見積り。実装時はライブラリ内部ワーク領域やローカル変数（スタック）を加味しても、現時点の設計規模では上限超過リスクは低い。文字列リテラルは`F()`マクロでフラッシュ配置を徹底するとさらに安全。

### 2-2. 関数一覧（役割の定義）

| 関数名 | 何をする関数か（1〜2行で） | 引数（入力） | 戻り値（出力） |
|:--|:--|:--|:--|
| `setup()` | ピンモード初期化、ディスプレイ初期化、初期数字表示 | なし | なし |
| `loop()` | 現在状態に応じた入力処理、回転更新、停止判定、結果判定を実行 | なし | なし |
| `startSpin()` | 全スロットを回転状態へ遷移し、停止フラグをリセット | なし | なし |
| `updateSpinningDigits()` | 未停止スロットの数字を50msごとに更新 | なし | なし |
| `handleButtonPress()` | 立ち下がり押下を検知し、1ループ1入力のみ有効化して該当スロット停止または再スタートを行う（リーチ演出中は本関数で入力処理しない）。同時押しは左→中央→右の走査順で最初の1入力を採用。1桁停止時は左/中央/右停止へ更新し、2桁停止時は残り1桁が回転中（currentStateは回転中扱い）、3桁停止時はisSpinning=falseで全停止・判定へ遷移 | なし | なし |
| `checkHitAndPlaySound()` | 全停止後に3桁一致を判定し、一致時は当選音再生状態へ遷移させる | なし | bool（アタリ可否） |
| `checkReachAndPlayCue()` | 2桁一致(リーチ)を判定し、任意機能ON時にリーチ音を再生 | なし | bool（リーチ可否） |
| `spinLastSlotForReach()` | リーチ演出専用の入力処理を行い、任意ボタン押下で未一致1桁の回転を開始（stoppedCount=2）し、対応ボタン押下で停止（stoppedCount=3）して再判定へ進める | なし | なし |
| `renderDisplay()` | 現在の3スロット値を4桁ディスプレイに反映 | なし | なし |
| `isDebouncedPress(pin)` | 指定ボタンのデバウンス済み押下イベントを返す | int pin | bool |

### 2-3. タイミング設計（delay vs millis）

| 処理 | 周期 | delay / millis | 理由 |
|:--|:--|:--|:--|
| スロット数字更新 | 50ms | **millis** | 回転中もボタン入力を取りこぼさないため |
| リーチ時の未一致1桁回転 | 50ms | **millis** | 任意ボタンで開始した未一致桁の回転を継続しつつ、停止入力を受け付けるため |
| ボタンデバウンス | 30〜50ms | **millis** | 長押し・チャタリング抑制を非ブロッキングで行うため |
| リーチ音再生 | 100〜300ms | **delay（短時間のみ）** | 演出区間であり、操作入力を受け付けないため |
| 当選音再生 | 音階ごとの短時間 | **delay（短時間のみ）** | 演出中は停止状態であり、仕様上問題が小さいため |

## 4. 異常系・エラー処理の方針

| 異常ケース | 検知方法 | 対応方針 |
|:--|:--|:--|
| ボタン長押し | ボタン押下状態が継続してもエッジが発生しない | 1回の押下イベントとしてのみ扱い、再入力は離した後に許可 |
| ボタン同時押し | 同一ループで複数ボタン押下を検知 | 先に検知した1入力のみ有効化し、残りは次ループで判定 |
| チャタリング | 前回有効入力から50ms未満の再入力 | 無効として捨てる |
| 想定外表示値 | 変数値が0〜9の範囲外 | `constrain`相当で0〜9へ丸め込み表示 |
| 任意機能の誤作動 | 任意機能フラグOFF時にリーチ演出呼び出し | リーチ判定をスキップし、通常の結果表示待機へ遷移 |
