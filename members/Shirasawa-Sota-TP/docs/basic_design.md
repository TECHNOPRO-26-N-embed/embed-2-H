## 1. システム全体像

### 1-1. ブロック図（入力 → 処理 → 出力）

```
【入力】                    【Arduino UNO R3】                【出力】
ボタン(左)  ──────────→                                ┌──→ 4桁ディスプレイ
ボタン(中)  ──────────→   スロット状態管理・乱数生成      ├──→ パッシブブザー
ボタン(右)  ──────────→   ボタン判定・表示更新制御        ┘
```

---

### 1-2. 状態遷移図（システムの「ふるまい」）

```
[電源ON]
	↓ 初期化完了
[待機(初期数字表示)] ─── 任意ボタン押下 ───→ [回転中]
│
├── 左ボタン押下 → [左停止]
├── 中央ボタン押下 → [左+中央停止]
└── 右ボタン押下 → [全停止・判定]
│
├── アタリ(3桁一致) → [当選音再生]
├── 任意機能ONかつリーチ(2桁一致) → [リーチ演出]
└── 上記以外 → [結果表示待機]

[リーチ演出] ─── リーチ音再生後、未一致1桁を自動回転 ───→ [全停止・判定]
[当選音再生] ─── 再生完了 ───→ [結果表示待機]
[結果表示待機] ─── 任意ボタン押下で再スタート ───→ [回転中]
```

**状態の意味を説明してください：**

| 状態名 | この状態のとき、何をしているか |
|:--|:--|
| 待機(初期数字表示) | 電源ON直後。各スロットに0〜9の任意数字を表示し、開始入力を待つ |
| 回転中 | 未停止のスロット数字を50ms周期(0.05秒)で更新する |
| 左停止 | 左スロットを固定し、中央・右スロットのみ回転する |
| 左+中央停止 | 左・中央スロットを固定し、右スロットのみ回転する |
| 全停止・判定 | 3スロット停止後に一致判定し、アタリ時のみブザーを鳴らす |
| リーチ演出（任意） | 2桁一致時にリーチ音を鳴らし、未一致1桁のみ自動回転する |
| 当選音再生 | 3桁一致時の当選音を再生する |
| 結果表示待機 | 判定結果を表示したまま、次ゲーム開始入力を待つ |

## 2. ソフトウェア設計

### 2-1. 使用するライブラリ

| ライブラリ名 | 用途 | インストール方法 |
|:--|:--|:--|
| TM1637Display | 4桁ディスプレイの表示制御 | ライブラリマネージャで検索 |
| なし（tone関数） | ブザー音再生 | Arduino標準機能 |

### 2-2. 関数一覧（役割の定義）

| 関数名 | 何をする関数か（1〜2行で） | 引数（入力） | 戻り値（出力） |
|:--|:--|:--|:--|
| `setup()` | ピンモード初期化、ディスプレイ初期化、初期数字表示 | なし | なし |
| `loop()` | 現在状態に応じた入力処理、回転更新、停止判定、結果判定を実行 | なし | なし |
| `startSpin()` | 全スロットを回転状態へ遷移し、停止フラグをリセット | なし | なし |
| `updateSpinningDigits()` | 未停止スロットの数字を50msごとに更新 | なし | なし |
| `handleButtonPress()` | 立ち下がり押下を検知し、該当スロット停止または再スタートを行う | なし | なし |
| `checkHitAndPlaySound()` | 全停止後に3桁一致を判定し、アタリなら当選音を再生 | なし | bool（アタリ可否） |
| `checkReachAndPlayCue()` | 2桁一致(リーチ)を判定し、任意機能ON時にリーチ音を再生 | なし | bool（リーチ可否） |
| `spinLastSlotForReach()` | リーチ時に未一致の1桁だけを自動回転して再判定へ進める | なし | なし |
| `renderDisplay()` | 現在の3スロット値を4桁ディスプレイに反映 | なし | なし |
| `isDebouncedPress(pin)` | 指定ボタンのデバウンス済み押下イベントを返す | int pin | bool |

### 2-3. タイミング設計（delay vs millis）

| 処理 | 周期 | delay / millis | 理由 |
|:--|:--|:--|:--|
| スロット数字更新 | 50ms | **millis** | 回転中もボタン入力を取りこぼさないため |
| リーチ時の未一致1桁回転 | 50ms | **millis** | 任意機能でも通常回転と同じ見た目・応答性を保つため |
| ボタンデバウンス | 30〜50ms | **millis** | 長押し・チャタリング抑制を非ブロッキングで行うため |
| リーチ音再生 | 100〜300ms | **delay（短時間のみ）** | 演出区間であり、操作入力を受け付けないため |
| 当選音再生 | 音階ごとの短時間 | **delay（短時間のみ）** | 演出中は停止状態であり、仕様上問題が小さいため |

## 3. ハードウェア設計

### 3-1. 部品一覧とピン接続

| 部品名 | 型番・種類 | Arduinoピン | 入力/出力 | 備考 |
|:--|:--|:--|:--|:--|
| ボタン(左) | プッシュボタン | D4 | 入力 | INPUT_PULLUP使用、押下でLOW |
| ボタン(中央) | プッシュボタン | D5 | 入力 | INPUT_PULLUP使用、押下でLOW |
| ボタン(右) | プッシュボタン | D6 | 入力 | INPUT_PULLUP使用、押下でLOW |
| 4桁ディスプレイ | TM1637系4桁7セグ | D2(CLK), D3(DIO) | 出力 | 右3桁をスロット表示に使用 |
| パッシブブザー | 圧電ブザー | D9 | 出力 | tone()で当選音を出力 |

> [!TIP]
> D0/D1はシリアル通信用に予約し使用しない。D2〜D6, D9のためピン競合なし。

### 3-2. 電気的考慮事項

| 用途 | 計算・メモ |
|:--|:--|
| ボタン入力安定化 | 内蔵プルアップを使うため外付け抵抗不要 |
| ブザー保護 | 小型パッシブブザーを想定。必要時は100〜220Ω直列抵抗を追加 |
| その他 | GND共通化。ボタンは片端GND、片端を各入力ピンに接続 |

## 4. 異常系・エラー処理の方針

| 異常ケース | 検知方法 | 対応方針 |
|:--|:--|:--|
| ボタン長押し | ボタン押下状態が継続してもエッジが発生しない | 1回の押下イベントとしてのみ扱い、再入力は離した後に許可 |
| ボタン同時押し | 同一ループで複数ボタン押下を検知 | 優先順位(左→中央→右)で1つのみ有効化 |
| チャタリング | 前回有効入力から50ms未満の再入力 | 無効として捨てる |
| 想定外表示値 | 変数値が0〜9の範囲外 | `constrain`相当で0〜9へ丸め込み表示 |
| 任意機能の誤作動 | 任意機能フラグOFF時にリーチ演出呼び出し | リーチ判定をスキップし、通常の結果表示待機へ遷移 |
