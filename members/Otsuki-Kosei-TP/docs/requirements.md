## 3. 機能要件

### 3-1. このガジェットができること（必須機能）
> ※ 「〜できる」の形で箇条書き。最低3つ書いてください。

- ランダムリズム生成
　　1 回のゲームで 1 回だけブザーを鳴らす
　　鳴動は 1 回のみ（複数回の点灯・連続パターンは行わない）
　　ブザーの鳴るリズムが1000msから5000msの間でランダムに変化させる

- 入力受付タイミング制御
   ブザー終了後 1000ms 待ってから入力受付開始
   入力受付時間は 最大 10000ms
   ボタン押下時間を計測し、押下タイミングを記録する

- 正誤判定ロジック
   プレイヤーの押下タイミングが
   ブザー鳴動時間 ±300ms 以内 → 正解
   300ms を超えたら不正解
   押しっぱなしが 7000ms を超えた場合は強制不正解
   7000ms超えた瞬間に即不正解判定

- LED フィードバック
   正解 → LED を 500ms 間隔で 3 回点滅
   不正解 → LED を 2000ms 点灯

- ゲーム再スタート
   判定後 2000ms 待機し、自動で開始状態に戻る

**状態遷移**

| 状態 | 説明 | 遷移条件 |
| --- | --- | --- |
| INIT | 初期化 | → READY |
| READY | 待機 | 1000ms 経過 → BUZZER |
| BUZZER | ランダム時間鳴動 | 鳴動終了 → WAIT_INPUT |
| WAIT_INPUT | 入力開始待ち | 1000ms 経過 → INPUT |
| INPUT | ボタン押下時間計測 | 押下完了 or 10000ms 経過 → JUDGE |
| JUDGE | 判定 | 判定完了 → RESULT |
| RESULT | LED 表示 | 表示終了 → READY |

- **ハード仕様（最小限の制約）**

| 部品 | ピン番号 | 備考 |
| --- | --- | --- |
| ブザー（Passive） | D8 | tone() 使用 |
| ボタン | D2 | INPUT_PULLUP |
| LED（赤） | D13 | 内蔵LEDでも可 |
| LED抵抗 | 220Ω | 標準値 |

- **比較するのは以下の2つ**
  ブザー鳴動時間（BUZZER_DURATION）
  プレイヤーのボタン押下時間（PRESS_DURATION）

 **判定条件**
   abs(PRESS_DURATION - BUZZER_DURATION) <= 許容誤差(ms)
   許容誤差は 300ms

 押下タイミング（リアクション時間）は判定に使わない


- **入力仕様（完全定義）**
  ボタンは 1 回押すだけ
  押した瞬間の時刻を記録
  押しっぱなしは 7000ms で強制不正解
  7000ms超えた瞬間に即不正解判定
  入力しなかった場合は不正解

- **入力の電気的仕様（完全定義）**
 ボタン
     INPUT_PULLUP を使用（アクティブ LOW）
     押したら LOW、離したら HIGH
     デバウンス：20ms

押下時間の計測方法
     LOW になった瞬間を start
     HIGH に戻った瞬間を end
     PRESS_DURATION = end - start

- **ランダム鳴動仕様（完全定義）**
  鳴動時間：1000〜5000ms
  重複可否：1 回のみなので問題なし
  鳴動間隔：なし（1 回だけ）
  鳴動回数：1 回固定

- **ランダム性の再現方法（完全定義）**
  andomSeed のシード元
     未接続アナログピン A0 の値を使用  
     randomSeed(analogRead(A0));


- **出力仕様（完全定義）**
 ブザー
  周波数：2000Hz
  鳴動時間：ランダム（1000〜5000ms）

 LED
  **LED 点滅仕様（完全定義）**
     正解：ON 500ms → OFF 500ms を 3回繰り返す
     不正解：ON 2000ms（点灯しっぱなし）

  **LED 表示が終わったら即 READY に戻る**
     RESULT → LED 表示（正解 or 不正解）
     LED 表示が終わったら 追加の待機時間なしで READY に遷移

- **受け入れ基準（テスト可能な数値条件）**
 時間系
     ブザー鳴動：1000〜5000ms の範囲
     入力受付開始：ブザー終了後 1000±50ms
     入力受付終了：開始から 10000±50ms
     押しっぱなし 7000ms 超 → 不正解確定
     7000ms超えた瞬間に即不正解判定

無入力
 ・定義
     入力受付時間（10000ms）内に押下が1度もない。

 10000ms 経過時点で不正解
 LED は不正解（2000ms 点灯）
 入力受付終了時点で PRESS_DURATION が未取得 → 不正解
     不正解 LED 表示（2000ms）も必ず行う
     
連打
 ・定義
     　入力受付中に 複数回押す（押して離すを2回以上） 行為。

 最初の押下開始〜最初の離脱のみを PRESS_DURATION として採用
 2回目以降の押下は 無視
 ただし、2回目以降の押下が 6000ms を超える押しっぱなし になった場合は不正解

チャタリング
 ・ 定義
    押下直後に HIGH/LOW が高速で揺れる現象。

 デバウンス時間 20ms を設ける
 20ms 以内の HIGH/LOW の変化は 無視
 20ms 以上安定した LOW → 押下開始
 20ms 以上安定した HIGH → 押下終了
 
 合否
     デバウンス後の PRESS_DURATION のみで判定
     チャタリングによる誤判定は起きないこと

押しっぱなし異常
 ・ 定義
     ボタンを押し続けて離さない。
     
 押下開始から 6000ms を超えた時点で不正解確定
 その瞬間に INPUT → JUDGE → RESULT に遷移
 LED は不正解（2000ms 点灯）

入力受付外の操作
 ・定義
     READY / BUZZER / WAIT_INPUT 中にボタンを押す。

 すべて無視
 PRESS_DURATION に記録しない
 状態遷移に影響しない
 
極端に短い押下（誤タッチ）
 ・ 定義
     押下時間が 20ms 未満（デバウンス未満）

 押下として扱わない
 無入力扱い（10000ms 経過で不正解）

極端に長い押下（仕様外）
 ・定義
     押下時間が 10000ms を超える（入力受付時間を超過）

 6000ms 時点で不正解確定（押しっぱなし異常扱い）

電気的ノイズによる誤検知
 ・定義
     HIGH/LOW が瞬間的に乱れる。

 デバウンス 20ms により無視されること
 ノイズで PRESS_DURATION が変化しないこと

LED の異常点灯
 ・定義
     LED が途中で消える・点灯しない。

 LED 表示時間が仕様どおりであること
 表示中にボタンを押してもゲーム状態に影響しないこと

ランダム値の偏り
 ・定義
     randomSeed が適切に機能せず、毎回同じ値が出る。

 A0 のノイズを使った randomSeed により
 連続10回の起動で同じ値が出ないこと

再スタート
 LED 表示終了後 即 READY に戻る
 READY 状態で 1000ms 待機後 BUZZER へ（ゲーム開始前の「間」）
