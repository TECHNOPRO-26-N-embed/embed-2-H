# テスト仕様書 — 反射神経を測ろうゲーム

<!-- 作成者: 有村 / 日付: 2026-05-27 / 対象: reaction_game.ino -->

## 0. 文書目的

本書は、[members/Arimura-Eishin-TP/src/reaction_game.ino](members/Arimura-Eishin-TP/src/reaction_game.ino) の動作確認手順と合否基準を定義する。
設計書（要件定義・基本設計・詳細設計）に対して、実装が期待どおり動作することを確認する。

---

## 1. テスト対象

- 対象コード: [members/Arimura-Eishin-TP/src/reaction_game.ino](members/Arimura-Eishin-TP/src/reaction_game.ino)
- 対象機能:
  - F01 反射測定
  - F02 ランダム待機
  - F03 表示処理
  - F04 記録更新
  - A01 フライング判定
  - A02 ブザー出力
  - A03 平均値表示
  - A04 ランキング更新（TOP5, RAM保持）
  - A05 ランキング表示（シリアルモニタ）

---

## 2. テスト環境

- ボード: Arduino UNO R3
- 使用ライブラリ: LiquidCrystal
- シリアルモニタ: 9600bps
- 入力:
    - Button(D2,INPUT_PULLUP)
- 出力:
    - LCD(D7-D12)
    - Passive Buzzer(D6)



---

## 3. 合否判定ルール

- 各テストケースで「期待結果」をすべて満たした場合を Pass とする。
- 1項目でも期待結果を満たさない場合は Fail とする。
- Fail の場合、再現手順・実測値・観測した表示内容を備考に記録する。

## 4. 単体テスト仕様

### 4-1. 状態遷移・入力判定

| ID | 観点 | 事前条件 | 手順 | 期待結果 |
|:--|:--|:--|:--|:--|
| UT-01 | 初期状態表示 | 電源ON直後 | 起動する | LCD 1行目に Press button、2行目に to start |
| UT-02 | IDLE→READY遷移 | UT-01完了 | ボタンを1回押す | READY / Get set... 表示へ遷移 |
| UT-03 | READYで早押し無効 | READY表示直後 | 0.8秒未満で押す | WAITへ遷移しない |
| UT-04 | READY→WAIT遷移 | READY表示中 | 0.8秒経過後に押す | WAIT... と Best: 表示へ遷移 |
| UT-05 | WAITでフライング判定 | WAIT表示中 | PUSH! 表示前に押す | MISS 表示へ遷移し失敗音が鳴る |
| UT-06 | WAIT→GO遷移 | WAIT表示中 | 押さずに待つ | PUSH! 表示 + 合図音再生 + GO状態へ遷移 |
| UT-07 | GOで反応計測 | PUSH! 表示中 | ボタンを押す | 結果画面に Time:xxxms 表示 |
| UT-08 | GOタイムアウト | GO状態 | 3秒以上押さない | TIMEOUT / Try again 表示後 RESULTへ遷移 |
| UT-09 | MISS→READY遷移条件 | MISS表示中 | 1.5秒未満で押す、次に1.5秒後押す | 前者は遷移しない、後者でREADYに遷移 |

### 4-2. 記録・表示ロジック

| ID | 観点 | 事前条件 | 手順 | 期待結果 |
|:--|:--|:--|:--|:--|
| UT-10 | Best初回更新 | 初回プレイ | 1回計測完了 | Best が Time と同値で表示 |
| UT-11 | Best更新あり | 2回目以降、より速い記録 | より短い時間で押す | Best が新しい短い値に更新 |
| UT-12 | Best更新なし | 2回目以降、遅い記録 | より長い時間で押す | Best は更新されない |
| UT-13 | 平均表示 | 1回以上計測済み | 結果画面遷移を観察 | Time/Best の後に Average が表示 |
| UT-14 | 表示時間 | 計測済み | ストップウォッチで測定 | Time/Best 約2秒、Average 約2秒 |
| UT-15 | RESULT→READY遷移 | RESULT表示中 | 表示シーケンス完了まで待つ | 自動で READY へ遷移 |

### 4-3. 効果音ロジック

| ID | 観点 | 事前条件 | 手順 | 期待結果 |
|:--|:--|:--|:--|:--|
| UT-16 | 合図音 | WAIT→GO成立 | PUSH! を待つ | 合図時に短音が鳴る |
| UT-17 | 失敗音 | WAIT中フライング | フライング操作 | MISS遷移時に失敗音が鳴る |
| UT-18 | ベスト更新メロディー | 記録更新可能状態 | ベスト更新を発生させる | 3音メロディーが鳴る |
| UT-19 | 音程確認 | 通常実行状態 | CUE/MISS/BESTをそれぞれ発生させる | CUE=1760Hz, MISS=330Hz, BEST=1319/1568/2093Hz相当 |
| UT-20 | ランキング更新順 | 計測成功を複数回実施 | 遅い記録→速い記録の順で入力する | シリアル表示が速い順（昇順）で並ぶ |
| UT-21 | ランキング件数上限 | 6回以上計測成功 | 記録を6件以上作る | シリアル表示は常に上位5件のみ |
| UT-22 | 非永続確認 | ランキング表示あり | 電源OFF→ONして起動 | シリアル表示が初期状態（No records yet）に戻る |

## 5. 結合テスト仕様（ユーザーシナリオ）

| ID | シナリオ | 手順 | 期待結果 |
|:--|:--|:--|:--|
| IT-01 | 通常1プレイ | 起動→READY→WAIT→GOで押下→RESULT | 一連の状態遷移が成立し、Time/Best/Averageが順に表示 |
| IT-02 | フライングから復帰 | 起動→READY→WAITでフライング→MISS→READY | MISS表示後、条件成立でREADYに戻れる |
| IT-03 | タイムアウト復帰 | GOで押さずにタイムアウト | TIMEOUT表示後RESULTを経てREADYに戻る |
| IT-04 | 連続プレイ | 5回連続プレイ | フリーズなし、Best/Averageが一貫して更新 |
| IT-05 | 最速更新演出 | 既存Bestより速い反応 | Best更新 + お祝いメロディー + 表示崩れなし |
| IT-06 | シリアルランキング連携 | 通常プレイを3回以上実施 | LCD表示と並行してシリアル確認 | 各成功プレイ後にランキングが更新表示される |

## 8. レビュー観点チェックリスト

- 状態遷移の全経路（通常/フライング/タイムアウト）を確認したか
- 表示時間（0.8秒/2秒/2秒/1.5秒）を実測したか
- ベスト更新あり/なしの両方を確認したか
- 連続操作時に入力取りこぼしやフリーズがないか
- LCD文字化けや表示崩れが再発していないか
- 最新配線（D2, D6, D7, D8, D9, D10, D11, D12）と仕様書記載が一致しているか
