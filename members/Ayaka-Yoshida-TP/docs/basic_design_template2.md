## 1. システム全体像

### 1-1. ブロック図（入力 → 処理 → 出力）

> ※ Arduinoを中心に、「何が入力で何が出力か」を整理します。  
> ASCII図または別紙の写真を貼り付けてください。

**記入例：**
```
【入力】               【Arduino UNO R3】          【出力】
超音波センサー ────→┐                       ┌──→ LED（赤・緑）
タクトスイッチ ────→┤    状態管理・制御      ├──→ パッシブブザー
                    └─────────────────────┘
```

**↓ 自分のシステムを書いてください**
```
【入力】                       【Arduino UNO R3】          【出力】
Rotary Encoder(回転/押下)    ─→┐                      ┌─→ Buzzer（音出力）
ボタン（切替）             ────→┤  状態管理・音制御     ├─→ LED（状態表示）
ポテンショメータ（音量）      ──→┘                  　  └─→
```

### 1-2. 状態遷移図（システムの「ふるまい」）

> ※ ガジェットが「どんな状態を持つか」を定義します。  
> 状態（○）と、状態が切り替わる条件（矢印の横）を書いてください。

**記入例：**
```
[電源ON]
  ↓ 初期化完了
[待機中] ─── センサー検知 ───→ [動作中] ─── 終了条件 ───→ [完了]
  ↑                                 │
  └──────── リセット ────────────────┘
```

**↓ 修正後の状態遷移図**
```
[電源ON]
  ↓ 初期化完了
[待機中] ─── エンコーダ操作 ───→ [設定変更中] ─── 決定/自動反映 ───→ [再生中]
  ↑                │                                      │
  └──── 停止/無操作 ┴──────── エラー検知 ───────────→ [エラー]
                      │
                      └── ポテンショメータ操作 ───→ [音量調整中] ─── 自動反映 ───→ [再生中]
```

| 状態名 | この状態のとき、何をしているか |
|:--|:--|
| 待機中 | 入力監視のみを行い、現在設定を保持する |
| 設定変更中 | エンコーダ回転/ボタン入力を読み取り、音程・音量・音色を更新する |
| 再生中 | 設定値に応じてブザー出力し、LEDで状態を表示する |
| エラー | 入力値異常時に音を停止し、警告表示を行う |
| 音量調整中 | ポテンショメータの値を読み取り、音量をリアルタイムで更新。再生中の音量に即時反映される。 |

## 2. ハードウェア設計

### 2-1. 部品一覧と接続ピン

> ※ 使う部品とArduinoのどのピンに繋ぐかを決めます。  
> デジタル（D2〜D13）/ アナログ（A0〜A5）/ 通信（I2C: A4/A5, SPI: D10〜D13）

| 部品名 | 型番・種類 | Arduinoピン | 入力/出力 | 備考 |
|:--|:--|:--|:--|:--|
| Rotary Encoder（CLK） | ロータリーエンコーダ | D2 | 入力 | 割り込み対応で回転検知 |
| Rotary Encoder（DT） | ロータリーエンコーダ | D3 | 入力 | CLKと組で方向判定 |
| Rotary Encoder（SW） | ロータリーエンコーダ押下 | D4 | 入力 | INPUT_PULLUP使用 |
| ボタン | プッシュボタン | D5 | 入力 | 音色切替用 |
| ポテンショメータ | 可変抵抗 | A0 | 入力 | 音量調整用 |
| LED（緑） | 5mm LED | D6 | 出力 | 220Ω抵抗と直列 |
| LED（赤） | 5mm LED | D7 | 出力 | エラー表示用 |
| Buzzer | パッシブブザー | D9 | 出力 | tone()で周波数制御 |

> [!TIP]
> **ピン競合チェック：** I2C（A4/A5）・SPI（D10〜D13）は複数部品で共用します。  
> デジタルピン（D2〜D9）は原則1部品1ピンです。D0/D1はシリアル通信に使われるため避けること。

### 2-2. 使用する抵抗・回路メモ

> ※ 抵抗値の計算や、注意が必要な接続を記録してください。

| 用途 | 計算・メモ |
|:--|:--|
| LED電流制限抵抗 | （5V − 2.0V）÷ 0.01A = 300Ω 程度、手持ち部品として220Ωを使用 |
| プルアップ/ダウン | エンコーダSW/ボタンはINPUT_PULLUPを使用し、押下時LOWで判定 |
| その他 | ブザーは大電流を流さない。GND共通を必ず確認する |

### 3-1. 使用するライブラリ

### 3-2. 関数一覧（役割の定義）

> ※ プログラムを小さな「処理の部品（関数）」に分割します。  
> **ここではコードを書かず、「何をする関数か」だけ日本語で書いてください。**  
> 具体的な実装は詳細設計書（detailed_design.md）で行います。

| 関数名 | 何をする関数か（1〜2行で） | 引数（入力） | 戻り値（出力） |
|:--|:--|:--|:--|
| `setup()` | Arduinoの初期化・ピンモード設定 | なし | なし |
| `loop()` | 状態に応じて各関数を呼び出すメインループ | なし | なし |

| `readEncoder()` | エンコーダの回転方向と増分を取得する | なし | int（-1/0/+1） |
| `handleButton()` | ボタン押下で音色モードを切り替える | なし | bool（切替有無） |
| `updateSoundParams(step)` | 入力に応じて音量・音程パラメータを更新する | int step | なし |
| `changeMelodyPattern()` | メロディパターンを切り替える | なし | なし |
| `saveSoundPattern()` | 設定した音パターンを保存する | なし | なし |
| `loadSoundPattern()` | 保存した音パターンを呼び出す | なし | なし |
| `playCurrentSound()` | 現在の設定でブザーを鳴らす | なし | なし |
| `checkError()` | 設定値の範囲外や入力異常を検知する | なし | bool |
| `showStatusLed(state)` | 状態に応じてLED表示を更新する | int state | なし |

| `adjustVolume()` | ポテンショメータの値を読み取り、音量を更新する | なし | なし |

### 3-3. タイミング設計（delay vs millis）

> ※ 複数の処理を同時に動かす場合、`delay()`を使うと他の処理が止まります。  
> 各タイマーが `delay` か `millis` かを事前に決めておきます。

| 処理 | 周期 | delay / millis | 理由 |
|:--|:--|:--|:--|

| エンコーダ読み取り | 10ms | **millis** | 操作遅延を抑えつつ安定検知するため |
| ボタンデバウンス | 20ms | **millis** | チャタリング抑制のため |
| 音出力更新 | 50ms | **millis** | 音の変化を滑らかに反映するため |
| ポテンショメータ読み取り | 50ms | **millis** | 音量調整をリアルタイムで反映するため |

## 4. 異常系・エラー処理の方針

> ※ 要件定義書の「非機能要件（3-4）」を具体化します。  
> 詳細な実装方法は詳細設計書で書きます。ここでは「どう検知し、どう対応するか」の方針だけ。

| 異常ケース | 検知方法 | 対応方針 |
|:--|:--|:--|
| チャタリング（ボタン） | 例：前回入力から50ms未満の入力を無効 | デバウンス処理を入れる |
| エンコーダ値の急変 | 前回差分が閾値を超える連続入力を検知 | 直前の安定値に戻し、次周期で再読込 |
| 音量/音程の範囲外 | 設定値 < 最小値 または > 最大値 | 範囲内にクリップし、LED赤点滅で通知 |
| ブザー出力異常 | 指定周波数が0以下または上限超過 | noTone()で停止し、エラー状態へ遷移 |

## 6. AIレビュー記録（基本設計版）

### Q1: 構造の妥当性確認
> 「この基本設計書をもとに Arduino UNO R3 でコードを書くとき、  
> ピン不足・ライブラリの競合・状態設計の漏れはありますか？」

**AIの回答（要約）：**
・UNO R3のピン数で実装可能。D0/D1を避ける設計は妥当。
・tone()使用時は他タイマ依存処理との競合を避ける必要あり。
・状態遷移に「エラー復帰条件」を明記すると実装が安定する。

**対応した内容：**
・エラー状態を追加し、停止/復帰の遷移方針を記載。
・入力はmillisベースで処理し、ブロッキングを回避。
・ピン割当を固定して競合リスクを低減。
