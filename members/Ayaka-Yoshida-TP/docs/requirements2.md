# 要件定義書 — 音程・音量チューニングシステム（改訂版）

<!-- 作成者: 吉田愛彩香 / 日付: 2026-05-27 / グループ: 2-H -->

## 1. プロジェクト概要

### 1-1. 作品タイトル
音程・音量チューニングシステム

### 1-2. 一言説明
エンコーダとポテンショメータで音を調整し、2種類のメロディを切替再生する装置。

### 1-3. 目的
- エンコーダで音程を細かく調整できること
- ポテンショメータの値を数値で確認できること
- パッシブブザーで確実に音を出せること

## 2. ユーザーと利用シーン

### 2-1. ユーザー
- 自分

### 2-2. 利用シーン
- 暇なときに音の変化を試す
- 組込み実装とデバッグの練習

## 3. 機能要件

### 3-1. 必須機能
- エンコーダで音程を1ステップずつ変更できる
- ポテンショメータの値を数値（raw, %, level）でシリアル表示できる
- ボタンで2種類のメロディを切替できる
- パッシブブザーでメロディを連続再生できる
- 音が鳴っている間は緑LED、鳴っていない間は赤LEDを点灯できる

### 3-2. メロディ仕様
- メロディA: ドレミファソ（半音オフセット: 0,2,4,5,7）
- メロディB: ソファミレド（半音オフセット: 7,5,4,2,0）
- ボタン押下ごとに A <-> B を切替

### 3-3. エンコーダ暴走抑制仕様
- ノイズガード時間未満のエッジは無視する
- 受理間隔ガード時間未満の連続ステップは無視する
- 短時間の連続無効入力が閾値を超えた場合はジャンプ検知フラグを立てる

### 3-4. テスト観点（シリアル出力要件）
- 全関数にデバッグ出力を配置する
- 全分岐に一意なトレース文字列（[COV] ...）を配置する
- 1回以上分岐通過したことがシリアルモニタで確認できる

### 3-5. 非機能要件
- 操作反映は0.5秒以内
- 異常時はブザー停止しエラー状態へ遷移
- デバッグ可能性を優先し、状態遷移・入力値・出力状態を定期表示

## 4. 使用部品

| 部品名 | 用途 |
|:--|:--|
| パッシブブザー | メロディ再生 |
| Rotary Encoder | 音程調整 |
| ボタン | メロディ切替 |
| ポテンショメータ | 音量調整 |
| LED（緑/赤） | 音の有無表示 |
| 抵抗・ジャンパ線 | 配線 |

## 5. スコープ外
- スマホ/PC連携はしない
- 外部スピーカー連携はしない
- 楽曲データの外部読み込みはしない

## 6. 受け入れ条件
- エンコーダを速く回しても値が一度に大きく飛びにくい
- シリアルモニタでポテンショメータ値を数値確認できる
- ボタンで2メロディが切替わる
- 音ON時は緑LED、音OFF時は赤LEDになる
- テスト時に全分岐通過ログを確認できる
